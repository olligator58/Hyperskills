Suppose you have a shared queue represented by an AtomicReferenceArray queue. Although we have not covered AtomicReferenceArray in the theory, we have talked about AtomicReference. An AtomicReferenceArray is an array of object references that can be updated atomically. (For more details and check the official Java documentation.)

Write a method enqueue that atomically adds an item to the end of the queue. If the queue is already full, the method must return false. Otherwise, it should add the item and return true.

You can assume that the queue size is 5 in the first test case.


Sample Input:
item1
item2
item3
item4
item5
item6

Sample Output:
true
true
true
true
true
false



Memory limit: 256 MB
Time limit: 5 seconds
